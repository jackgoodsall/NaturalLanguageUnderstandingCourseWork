COMP34812 Coursework · Evidence Detection

Evidence Detection With a BiLSTM Baseline and a Transfer-Initialised DeBERTa Ensemble

We model claim-evidence relevance as a binary sequence-pair classification problem. The final coursework submission uses one non-transformer system (`Solution B`) and one transformer system (`Solution C`), both evaluated on the official closed-track train/dev data and locked through explicit run records.

Task Summary

Given a claim and a candidate evidence snippet, predict whether the evidence is relevant.

  • Track: `Evidence Detection (ED)`
  • Setting: closed-track coursework data only
  • Submission pair: one `B` method and one `C` method

Solution B

ESIM-Style BiLSTM

Goal: provide a strong non-transformer baseline that still models claim-evidence interaction rather than classifying each sequence independently.

1
EmbeddingLowercase, lightweight cleanup, FastText 300d vectors via gensim.
2
Shared BiLSTM EncodingClaim and evidence are encoded with the same sequence encoder.
3
Co-AttentionCross-attention aligns both sequences and exposes token-level interaction.
4
Composition + PoolingEnhanced representations pass through a second BiLSTM and mean/max pooling.
5
ClassificationMLP head outputs the binary relevance score.

The locked `B` path is threshold-sensitive. The default threshold `0.50` underperformed the tuned operating threshold `0.60`, indicating that calibration mattered materially for minority-class behaviour.

Error Analysis

Observed Failure Modes

  • Solution B: over-predicts the positive class at the default threshold, which hurts both macro F1 and MCC.
  • Solution C: still makes `451` false positives and `207` false negatives on dev at the locked threshold.
  • Many remaining errors appear to come from evidence that is semantically related to the claim but not actually strong enough to count as relevant evidence.
Solution C

Transfer-Initialised DeBERTa-v3 Ensemble

Goal: maximise transformer performance while keeping the path stable, reproducible, and clearly measurable.

1
Base ModelStart from MoritzLaurer/DeBERTa-v3-base-mnli-fever-anli.
2
Pair EncodingCross-encoder on claim/evidence pairs with max length 256.
3
Stable TrainingFloat32 path, `lr=1e-5`, `epochs=3`, `weight_decay=0.01`, warmup `0.06`.
4
EnsembleTrain seeds `42`, `52`, `62` and average probabilities.
5
Locked PathUse default threshold `0.50` for the submission system.

Transfer initialisation produced a much larger gain than continuing broad tuning on the plain baseline. A three-seed ensemble improved robustness, while pushing to five seeds gave weak returns.

Limits and Risks

What This Work Does Not Solve

  • The dataset is imbalanced, so headline accuracy can hide minority-class failures.
  • Neither system provides explanatory rationales for why evidence was judged relevant.
  • False positives can surface irrelevant evidence; false negatives can hide useful evidence.
  • The closed-track setting prevents use of external retrieval or evidence-ranking data.
